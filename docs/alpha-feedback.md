# MindCanary Linux Alpha Feedback

Thank you for testing MindCanary. We are trying to learn whether the local app
is understandable, trustworthy, and useful before it has weeks of history.

Participation is voluntary and is not medical care or a clinical study. The app
has no account or telemetry and does not send us your records. You may pause,
delete, or stop using it at any time. Only feedback you intentionally send
leaves your device.

Choose a participant code that does not identify you, and use the same code in
each reply:

```text
Participant code:
```

## Please Do Not Send

Do not send exports, database files, recovery secrets, browser extension
storage, or screenshots containing private check-ins or activity patterns.
MindCanary support should not need them for an initial report.

## First Use

1. Did installation and first launch work without terminal help?
2. Was it clear that check-ins, browser context, and computer activity are all
   optional?
3. Could you save a check-in and find it again after reopening the app?
4. Did any wording feel clinical, alarming, judgmental, or guilt-inducing?
5. Did the app provide any value before it had a personal baseline?
6. What was confusing or unexpectedly difficult?

## Day 3

1. What made you reopen MindCanary, if anything?
2. Did check-ins feel useful, tiring, or encourage excessive checking?
3. Did any connector feel more invasive than expected?

## Day 7

1. Did a chart or description help you notice something useful?
2. Did any wording feel judgmental, clinical, or more certain than the data?
3. Did you pause or stop using anything? What prompted that choice?

## Day 21

1. Does MindCanary have a place in your routine now? Why or why not?
2. Which parts would you keep, remove, or change?
3. Would you trust it enough to keep using local connectors?
